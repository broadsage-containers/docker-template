# 💬 Communication Guide

## 🚀 Quick Contact

| 🎯 **What You Need** | 📍 **Where to Go** | ⏱️ **Response Time** |
|---------------------|-------------------|---------------------|
| 🐛 **Report a Bug** | [GitHub Issues](https://github.com/broadsage/containers/issues) | 2-3 business days |
| ❓ **Ask Questions** | [GitHub Discussions](https://github.com/broadsage/containers/discussions) | Community-driven |
| 🔒 **Security Issue** | [security@broadsage.com](mailto:security@broadsage.com) | Within 24 hours |
| 🤝 **Partnership/Business** | [opensource@broadsage.com](mailto:opensource@broadsage.com) | 1-2 business days |
| ⚖️ **Governance/Maintainer** | [tsc@broadsage.com](mailto:tsc@broadsage.com) | 1-2 business days |

## 🏠 GitHub Channels

### 🐛 Issues

**Perfect for:** Specific problems that need tracking

✅ Bug reports with clear reproduction steps  
✅ Feature requests with detailed requirements  
✅ Documentation improvements  

### 💭 Discussions

**Perfect for:** Open conversations and community help

✅ "How do I..." questions  
✅ Implementation advice and best practices  
✅ Community announcements and updates  
✅ Sharing your container use cases

## 🎯 Our Approach

We keep communication **simple and GitHub-focused**:

- **🐙 GitHub-First**: Everything lives where the code lives
- **👥 Community-Driven**: Community helps each other with maintainer guidance  
- **🌟 Transparent**: Public decisions (except security/privacy matters)

## 🎯 Quick Tips

| 💡 **Do** | ❌ **Don't** |
|----------|-------------|
| Search existing issues first | Create duplicates |
| Use clear, descriptive titles | Be vague or generic |
| Provide reproduction steps | Just say "it doesn't work" |
| Be patient with volunteers | Expect immediate responses |
